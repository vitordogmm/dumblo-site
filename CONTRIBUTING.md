# Contribuindo com o Dumblo Site

Obrigado por querer contribuir! Este guia explica como abrir issues e enviar PRs.

## Fluxo de trabalho
- Fork o repositório e crie uma branch a partir de `main`.
- Use mensagens de commit claras (ex.: `feat:`, `fix:`, `docs:`). Opcional: [Conventional Commits](https://www.conventionalcommits.org/).
- Envie um Pull Request descrevendo o que foi alterado.

## Reportando problemas
- Antes de abrir uma issue, verifique se já existe algo relacionado.
- Inclua passos de reprodução, o comportamento esperado e capturas de tela quando possível.

## Estilo
- HTML/CSS/JS simples, sem dependências externas.
- Evite mudanças não relacionadas ao objetivo do PR.

## Roadmap
- Sugestões são bem-vindas via issues. Explique o impacto e o valor para os usuários.

## Contato
- Use as issues para discussões públicas.
- Para assuntos sensíveis, siga `SECURITY.md`.
